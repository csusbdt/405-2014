Overview

The purpose of this assignment is to learn how to deploy a Web application using the Heroku cloud service platform.

This assignment requires research and self-study. If you get to this assignment early enough, I can help you solve some of the problems you encounter.

I covered deployment to Heroku in the pre-requisite course Web Page Programming. If you did not take this course, then you will need to put extra time into learning how to deploy onto Heroku. Even if you did take the course, there are likely many additional things you need to learn in order to deploy a version of your gift application to it.

Assignment Folder

There is no special folder for this assignment. You will need to create a Procfile, package.json file, and node_modules folder in the root level of your repository. I also advise to create file .slugignore to omit unnecessary files from the Heroku application slug.

To complete this assignment, you need a way for your code to distinguish between local and remote deployments. I recommend that you set environmental variables to accomplish this. You will need to use Node.js's process object to do this, which was covered in the scope assignment.
